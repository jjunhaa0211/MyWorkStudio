# Doffice Regression Checklist

이 문서는 릴리스 후보, 큰 리팩터링, 세션/플러그인/런타임 관련 변경 후에 사용하는 운영 체크리스트입니다.

## 1. 자동 게이트

- [ ] `./Scripts/smoke-test.sh`
- [ ] GitHub Actions `Build & Release`가 동일 커맨드로 성공
- [ ] 스모크 로그에 `[SmokeTest] Smoke test passed`가 출력됨

## 2. 런치와 기본 상태

- [ ] 앱이 크래시 없이 실행된다
- [ ] 메인 윈도우가 정상 크기로 보이고 빈 화면이 아니다
- [ ] 다크/라이트 토글 후 주요 표면, 버튼, 검색 필드, 배지가 모두 읽기 좋다
- [ ] 설정 창, 액션 센터, 커맨드 팔레트가 각각 열린다
- [ ] 앱 재실행 후 마지막 기본 상태가 깨지지 않는다

## 3. 세션 탐지와 기본 사용 시나리오

- [ ] 새 세션 생성이 성공한다
- [ ] `⌘R` 또는 수동 새로고침 후 세션 목록이 중복 증가하지 않는다
- [ ] 세션 전환, 닫기, 다시 열기가 정상 동작한다
- [ ] 온보딩 미완료 상태와 완료 상태 모두에서 메인 화면 진입이 정상이다
- [ ] 토큰/상태 바 영역이 비정상적으로 비거나 깨지지 않는다

## 4. SSH 시나리오

- [ ] SSH 프로필 추가가 저장된다
- [ ] SSH 프로필 수정 후 재실행해도 값이 유지된다
- [ ] 원격 작업 디렉터리에 공백이나 `'`가 있어도 명령 문자열이 깨지지 않는다
- [ ] SSH 프로필 삭제가 정상 반영된다

## 5. tmux 시나리오

- [ ] `tmux` 설치 환경에서 세션 생성이 된다
- [ ] 앱 재시작 후 기존 `tmux` 세션을 다시 찾는다
- [ ] 세션 목록에 `doffice-` prefix 외 외부 세션이 섞여 보이지 않는다
- [ ] 세션 종료 후 orphan 세션이 남지 않는다

## 6. 플러그인 설치 시나리오

- [ ] 로컬 디렉터리 플러그인 설치가 성공한다
- [ ] `CLAUDE.md`만 있는 최소 플러그인도 정상 등록된다
- [ ] 빈 디렉터리나 잘못된 경로는 실패로 처리된다
- [ ] 다운로드 기반 설치가 성공하고 목록에 반영된다
- [ ] 깨진 zip/tar는 실패로 처리되고 임시 설치 디렉터리가 남지 않는다
- [ ] 설치 후 플러그인 비활성화/재활성화가 정상 동작한다

## 7. 디자인 시스템 시각 점검

- [ ] `DesignSystem` 카탈로그에서 사이드바, 히어로, 섹션 카드가 모두 렌더링된다
- [ ] 주요 버튼의 prominent / neutral / compact 변형이 모두 자연스럽다
- [ ] 검색 필드, 상태 배지, 메트릭 카드에 레이아웃 깨짐이 없다
- [ ] 다크/라이트 모드 모두 텍스트 대비가 충분하다
- [ ] 카드, 패널, 인풋의 하이라이트와 그림자가 과하지 않고 일관적이다
